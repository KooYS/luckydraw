# Project Brief: LuckyDraw

## Executive Summary

KPOP 이벤트 현장에서 사용하는 종이 기반 럭키드로우 시스템을 웹 애플리케이션으로 대체하는 프로젝트입니다. **관리자가 조작하고 고객은 화면으로 결과를 확인하는 키오스크/POS 스타일**의 시스템으로, 결제 후 즉시 럭키드로우를 진행하고 결과지를 출력하여 현장에서 상품을 전달합니다. 매 이벤트마다 물리적 럭키드로우 제작에 드는 비용과 시간을 절감하고, 고객에게 더 나은 시각적 경험을 제공합니다.

---

## Problem Statement

### 현재 상태와 Pain Points
- 매 KPOP 이벤트마다 종이 럭키드로우를 물리적으로 제작해야 함
- 제작 비용(인쇄, 디자인, 포장)이 이벤트 예산의 상당 부분을 차지
- 재고 관리의 어려움 (남거나 부족한 경우 발생)
- 이벤트별 커스터마이징에 추가 시간과 비용 소요
- 결과 집계와 당첨자 관리가 수동으로 진행됨

### 문제의 영향
- 이벤트당 럭키드로우 제작 비용 발생 (수량에 비례)
- 제작 리드타임으로 인한 기획 유연성 저하
- 환경적 부담 (일회용 종이 사용)

### 기존 솔루션의 한계
- 일반적인 룰렛/추첨 앱은 KPOP 이벤트의 특수한 니즈(포토카드 컨셉, 팬 경험 등)를 충족하지 못함
- 브랜딩 커스터마이징이 제한적

---

## Proposed Solution

KPOP 이벤트에 특화된 웹 기반 디지털 럭키드로우 시스템을 구축합니다.

### Core Concept
- 관리자용 태블릿(아이패드) 기반 웹 애플리케이션
- 관리자가 조작하고 고객은 화면으로 결과 확인 (키오스크/POS 스타일)
- 이벤트/아티스트별 테마 커스터마이징 지원
- 결제 후 즉시 럭키드로우 실행 및 현장 상품 전달

### Key Differentiators
- KPOP 팬덤 문화에 맞춘 UX/UI (포토카드 뒤집기, 티켓 스크래치 등)
- 이벤트 운영자용 관리 대시보드
- 재사용 가능한 템플릿 시스템으로 빠른 이벤트 설정

---

## Target Users

### Primary User Segment: 이벤트 운영자 (관리자)
- **프로필:** 이벤트 현장 운영 스태프
- **행동:** 아이패드로 럭키드로우 시스템 조작, 결제 확인 후 실행
- **니즈:** 빠르고 간편한 조작, 안정적인 시스템, 쉬운 이벤트 설정
- **목표:** 효율적인 현장 운영, 고객에게 즐거운 경험 제공

### Secondary User Segment: 이벤트 참가자 (팬/고객)
- **프로필:** KPOP 팬, 10대~30대
- **행동:** 결제 후 관리자 화면을 보며 럭키드로우 결과 확인
- **니즈:** 재미있는 시각적 경험, 공정한 추첨, 즉시 결과 확인
- **목표:** 원하는 상품/포토카드 당첨, 이벤트 참여의 즐거움

### Tertiary User Segment: 이벤트 기획자 (백오피스)
- **프로필:** 이벤트 기획/마케팅 담당자
- **행동:** 이벤트 전 설정, 상품/확률 구성, 결과 데이터 확인
- **니즈:** 쉬운 이벤트 설정, 결과 리포트, 재사용 가능한 템플릿
- **목표:** 효율적인 이벤트 기획, 비용 절감, 데이터 기반 의사결정

---

## Goals & Success Metrics

### Business Objectives
- 럭키드로우 관련 비용 50% 이상 절감 (첫 해 기준)
- 이벤트 설정 시간 80% 단축 (종이 제작 대비)
- 모든 신규 이벤트에서 디지털 럭키드로우 사용

### User Success Metrics
- 참가자 참여 완료율 95% 이상
- 평균 참여 시간 30초 이내
- 참가자 만족도 4.0/5.0 이상

### Key Performance Indicators (KPIs)
- **이벤트당 비용:** 종이 대비 절감액 추적
- **참여율:** 이벤트 참석자 대비 럭키드로우 참여자 비율
- **시스템 안정성:** 이벤트 중 다운타임 0건

---

## MVP Scope

### Core Features (Must Have)
- **현장 운영 화면:** 관리자가 조작하는 럭키드로우 실행 화면 (아이패드 최적화)
- **고객용 결과 화면:** 인터랙티브한 럭키드로우 애니메이션 및 결과 표시
- **관리자 대시보드:** 이벤트 생성, 상품/확률 설정, 재고 관리
- **당첨 로직:** 확률 기반 추첨, 상품별 수량 제한 및 실시간 재고 차감

### Out of Scope for MVP
- 복잡한 애니메이션 (포토카드 뒤집기 등은 Phase 2)
- 다국어 지원
- 외부 결제 연동
- 소셜 미디어 공유 기능
- 푸시 알림
- 상세 통계/분석 대시보드

### MVP Success Criteria
- 1회 이상의 실제 이벤트에서 성공적으로 운영
- 안정적인 연속 운영 (하루 수백 건 럭키드로우 처리)
- 이벤트 운영자가 30분 이내 신규 이벤트 설정 가능
- 현장 스태프가 5분 내 조작법 습득 가능

---

## Post-MVP Vision

### Phase 2 Features
- 다양한 럭키드로우 테마/애니메이션 (포토카드, 스크래치, 룰렛 등)
- 이벤트 템플릿 라이브러리
- 참가자 데이터 수집 (선택적 설문)
- 상세 분석 대시보드

### Long-term Vision
- KPOP 이벤트 통합 플랫폼으로 확장 (출첵, 포토존, 굿즈 예약 등)
- 외부 기업/기획사에 SaaS로 제공
- 오프라인-온라인 연계 이벤트 지원

### Expansion Opportunities
- 팬사인회, 콘서트, 팝업스토어 등 다양한 이벤트 유형 지원
- API 제공으로 타 시스템 연동
- 화이트라벨 솔루션

## Technical Considerations

### Platform Requirements
- **Target Platforms:** 태블릿 웹 (아이패드 Safari 최적화, 데스크톱 호환)
- **Browser Support:** Safari (iPad), Chrome (관리자 대시보드용)
- **Performance Requirements:** 초기 로딩 3초 이내, 추첨 결과 즉시 응답, 애니메이션 60fps

### Technology Preferences
- **Frontend:** React 또는 Next.js (빠른 개발, 좋은 모바일 경험)
- **Backend:** Node.js 또는 Python (확장성 고려)
- **Database:** PostgreSQL 또는 MongoDB
- **Hosting/Infrastructure:** Vercel 또는 AWS (auto-scaling 필요)

### Architecture Considerations
- **Repository Structure:** Monorepo (frontend + backend + admin)
- **Service Architecture:** 단순 시작, 필요시 분리
- **Integration Requirements:** QR 코드 생성, 이미지 업로드
- **Security/Compliance:** 참가자 정보 최소 수집, HTTPS 필수

## Constraints & Assumptions

### Constraints
- **Budget:** 초기 개발은 내부 리소스 활용, 인프라 비용 최소화
- **Timeline:** 다음 주요 이벤트 전 MVP 완성 목표
- **Resources:** 소규모 개발팀
- **Technical:** 이벤트 현장 네트워크 환경 불안정 가능성

### Key Assumptions
- 이벤트 현장에 아이패드와 인터넷 연결 환경 확보 가능
- 현장 스태프가 간단한 터치 인터페이스 조작 가능
- 결제는 별도 시스템(현금/카드)으로 처리하고, 럭키드로우는 결제 확인 후 진행
- 초기에는 사내 이벤트에서만 사용

## Risks & Open Questions

### Key Risks
- **현장 네트워크 불안정:** 이벤트 현장 인터넷 품질에 따른 서비스 장애 → 오프라인 폴백 또는 로컬 캐싱 고려
- **기기 장애:** 아이패드 배터리 방전, 앱 크래시 등 → 빠른 복구 방안 필요
- **재고 불일치:** 실제 상품 재고와 시스템 재고 불일치 → 실시간 동기화 및 수동 조정 기능 필요

### Clarified Decisions
- **참가자 인증:** 익명 방식. 별도 인증 없이 결제 후 관리자가 럭키드로우 실행
- **상품 수령:** 현장 즉시 수령. 관리자가 결과 확인 후 직접 상품 전달
- **운영 플로우:** 기존 종이 럭키드로우를 관리자 아이패드의 웹앱으로 대체. 소비자는 화면에서 수량과 인터랙티브한 결과만 확인하고 상품 수령
- **다중 참여:** 한 이벤트에서 여러 사람이 여러 번 참여 가능 (결제 건당 1회 럭키드로우)

- 이벤트 현장 네트워크 환경 테스트
