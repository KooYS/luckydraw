# LuckyDraw MVP 스프린트 계획

**방식:** 3-5일 단위 미니 스프린트
**개발자:** 1명
**목표:** 최대한 빠른 MVP 출시

---

## Sprint 0: 프로젝트 셋업 (1-2일)

### 목표
개발 환경 구성 및 기술 스택 확정

### Tasks
- [ ] 기술 스택 최종 확정 (Frontend/Backend/DB)
- [ ] 프로젝트 초기화 및 기본 구조 생성
- [ ] 개발 환경 설정 (lint, prettier, etc.)
- [ ] DB 스키마 설계
  - Event (이벤트)
  - Product (상품/당첨품)
  - DrawResult (당첨 기록)
- [ ] 기본 라우팅 구조 설정

### 완료 조건
- 프로젝트가 로컬에서 실행됨
- DB 연결 확인
- 기본 페이지 라우팅 동작

---

## Sprint 1: 핵심 로직 (3-4일)

### 목표
럭키드로우의 핵심인 당첨 로직 완성

### Tasks
- [ ] 확률 기반 당첨 로직 구현
- [ ] 상품별 재고 관리 (수량 제한, 실시간 차감)
- [ ] API 엔드포인트 구현
  - POST /api/draw - 럭키드로우 실행
  - GET /api/events/:id - 이벤트 정보 조회
  - GET /api/events/:id/products - 상품 목록 및 재고
- [ ] 당첨 결과 저장

### 완료 조건
- API로 럭키드로우 실행 시 확률에 따라 상품 당첨
- 재고 소진 시 해당 상품 당첨 불가
- 당첨 기록이 DB에 저장됨

---

## Sprint 2: 운영 화면 (3-4일)

### 목표
현장에서 사용할 관리자/고객 화면 구현

### Tasks
- [ ] 관리자용 럭키드로우 실행 화면
  - 이벤트 선택
  - 럭키드로우 실행 버튼
  - 현재 재고 표시
- [ ] 고객용 결과 화면
  - 럭키드로우 애니메이션 (간단한 버전)
  - 당첨 결과 표시 (상품명, 이미지)
  - 꽝 화면
- [ ] 아이패드 Safari 최적화

### 완료 조건
- 관리자가 버튼 클릭으로 럭키드로우 실행 가능
- 고객이 화면에서 결과 애니메이션 확인 가능
- 아이패드에서 정상 동작

---

## Sprint 3: 관리 기능 (3-4일)

### 목표
이벤트와 상품을 설정할 수 있는 관리자 대시보드

### Tasks
- [ ] 이벤트 CRUD
  - 이벤트 생성/수정/삭제
  - 이벤트 활성화/비활성화
- [ ] 상품 관리
  - 상품 추가/수정/삭제
  - 당첨 확률 설정
  - 재고 수량 설정/조정
- [ ] 재고 현황 대시보드
  - 실시간 재고 모니터링
  - 당첨 내역 확인

### 완료 조건
- 새 이벤트 생성 및 상품 설정 가능
- 재고 수동 조정 가능
- 당첨 내역 조회 가능

---

## Sprint 4: 마무리 (2-3일)

### 목표
실제 이벤트 투입 가능한 상태로 완성

### Tasks
- [ ] 전체 플로우 E2E 테스트
- [ ] 버그 수정
- [ ] UI/UX 다듬기
- [ ] 배포 (Vercel 또는 선택한 플랫폼)
- [ ] 실제 이벤트 시뮬레이션 테스트

### 완료 조건
- 프로덕션 배포 완료
- 실제 이벤트 시나리오 테스트 통과
- 현장 스태프 사용 가능한 상태
